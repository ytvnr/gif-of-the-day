Realtime Database

  • Cloud-hosted NoSQL database

  • Just one big JSON object that the developers can manage in realtime

  • A single API that provides both current value and any update to the data

Firestore

  • A NoSQL document database to store, sync and query data

  • More structured approach, data keept inside objects called documents, which are grouped into collections.

  • Different from Realtime Database (see here which database you need)

  • Multi-region database (data automatically copied to various regions)

Queries are created by chaining together one or more filter methods.

firebase.firestore.collection('teams')
.where('organization', '==', this.db.collection('organizations').doc(organizationId))
.get();

  • Rollback with one click

  // setup the reactive teams property
  data: () => ({ teams: [] }),

  firestore: {
    teams: db.collection('teams'),
  },
})

Store with Firebase üî•

    export default new Vuex.Store({
        // ...
        actions: {
            getUserMetadata({ commit }, userId) {
                firebase
                    .firestore()
                    .collection('usersMetadata')
                    .doc(userId)
                    .onSnapshot(snapshot => {
                        this.userMetadata = snapshot.data();
                        commit(
                            'setAssignedTeamId',
                            this.userMetadata?.assignedTeamId
                        );
                    });
            }
        }
    }

GitHub Actions

Allow to automate, customize and execute your software development workflows in your repository.

Some wording

Action = Individual task that you combine as steps to create a job. The smallest building block of a workflow.

Step = An individual task that can run command or actions

Job = A set of steps that execute on the same runner.

Workflow = A configurable automated process that you can set up in your repo. Composed of one or many jobs and can be scheduled or triggered by an event.

More: Core concepts for GitHub Actions
